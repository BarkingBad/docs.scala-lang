    Source links

    Source links are used to point to source code at some remote repository like github or bitbucket. Hopefully, the new syntax is almost superset of the old syntax. To migrate to the new scaladoc syntax, make sure that you donâ€™t use any of these variables: â‚¬{TPL_OWNER} or â‚¬{FILE_PATH_EXT}. Otherwise you have to rewrite your source link, using either other variables or you can use new syntax, about which you can read more at dotty docs Note that new syntax let you specify prefixes of your files paths to match specific url in case your sources are scattered in different directories or even different repositories.

    External mappings

    This setting is a generalized form of the old settings for javadoc/scaladoc.

    Example external mapping is:

    -external-mappings:.*scala.*::scaladoc3::http://dotty.epfl.ch/api/,.*java.*::javadoc::https://docs.oracle.com/javase/8/docs/api/

    A mapping is of the form â€˜<regex>::[scaladoc3|scaladoc|javadoc]::<path>â€™. You can supply several mappings, separated by commas, as shown in the example.
